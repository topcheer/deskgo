# DeskGo v0.1.2 发布说明

## 发布定位

`v0.1.2` 的目标，是把最近一轮 Relay、安装脚本与 Windows 启动修复真正变成“可安装、可下载、可复现”的正式版本，让 GitHub 在线安装脚本与用户手动下载到的 Desktop CLI 二进制，终于和 `master` 分支上的行为保持一致。

这次补丁版本主要聚焦三件事：

- 让跨平台自动运行安装脚本达到可发布状态
- 修复 Windows 启动链路与 PowerShell 兼容性问题
- 统一 Relay / Session 处理逻辑，降低手动运行 CLI 时的脆弱性

## 亮点

### 1. 跨平台自动运行安装脚本进入正式发布面

- 新增引导式与非引导式安装脚本：
  - macOS / Linux：`deskgo-autostart.sh`
  - Windows PowerShell：`deskgo-autostart.ps1`
- 安装脚本会下载与当前平台匹配的 Desktop CLI 产物，并校验 `SHA256SUMS.txt`
- 各平台统一采用“当前用户桌面会话级”的自动启动方式：
  - Windows：计划任务
  - macOS：LaunchAgent
  - Linux：XDG Autostart 或可选 `systemd --user`

### 2. Windows 自动运行更安全、更可诊断

- 修复了 PowerShell 编码问题与 Windows PowerShell 5.1 兼容性问题
- 生成的 `run-desktop.ps1` 启动脚本现已具备 BOM，可被 Windows PowerShell 正确解析
- Windows 登录自启动链路现在具备：
  - 隐藏 PowerShell 启动
  - 登录后延迟启动
  - 失败重试日志
  - 重装 / 卸载前主动清理旧任务与旧进程
- 启动日志位于 `%LOCALAPPDATA%\DeskGo\logs\desktop.log`

### 3. 手动连接 Relay 时更宽容

- Desktop CLI 现在接受 `http(s)://` 与 `ws(s)://` 两种 Relay 地址输入，并会在拨号前自动规范化
- Session ID 在客户端、Relay、网页和安装脚本生成的配置中统一按大小写不敏感处理
- Relay / Web URL 推导逻辑会正确保留自定义路径前缀，避免浏览器访问地址被错误裁剪

### 4. 在线安装流程与 GitHub 分发逻辑对齐

- 首页、README 与自动运行文档现已统一以 GitHub Raw 作为安装脚本入口
- 通过 no-cache / cache-busting 片段减少陈旧脚本缓存问题
- Release 产物中也直接附带安装脚本，保证在线文档与 Release 内容一致

## 验证

发布前已完成以下验证：

- `go test ./cmd/client ./cmd/relay ./internal/relay`
- `go build -tags desktop -o /tmp/deskgo-client-desktop ./cmd/client`
- `GOOS=windows GOARCH=amd64 CGO_ENABLED=0 go build -tags desktop -o /tmp/deskgo-client-windows.exe ./cmd/client`
- `bash -n scripts/deskgo-autostart.sh`
- `pwsh -NoProfile -File scripts/deskgo-autostart.ps1 install -RelayServer http://192.168.31.105:8082 -Session WiN11Studio -Version v0.1.1 -NonInteractive -DryRun`
- 验证安装脚本会把 Relay 地址正确规范化为 `ws://` / `wss://`
- 验证生成的 Windows `run-desktop.ps1` 启动器可被 PowerShell 正确解析

## 已知限制

- Windows 自动运行目前仍仅支持 `jpeg`；Windows 的 H.264 自动运行编码路径尚未实现
- Linux 输入控制仍依赖 X11 / XWayland
- Relay 仍建议部署在反向代理、隧道或其它访问控制层之后
