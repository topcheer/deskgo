# DeskGo v0.1.1 Release Note（草案）

## 发布定位

`v0.1.1` 是首发版本之后的稳定性修复发布，重点补齐真实部署里暴露出的 Relay、Web 查看器与 Desktop CLI 串流生命周期问题。

这次发布主要解决两类问题：

- 浏览器接入后无法正确触发 CLI 开始串流
- 所有 Web 查看器断开后 CLI 仍持续推流，浪费带宽

同时保留并纳入此前已经完成但尚未进入正式 Release 的能力：

- Desktop CLI 通过 `-proxy` / 环境变量连接 Relay
- Demo 地址与下载入口体验改进
- macOS 下载链接回退优化

## 亮点

### 1. 修复 Web 端会话注入导致的 `start_capture` 失效

- 修正会话页脚本中的模板注入方式，避免 `session_id` 被双重引用
- 浏览器现在会连接到真实会话 ID，而不是带引号的错误会话
- Relay 可以重新正确通知 CLI 发送 `start_capture`

### 2. 最后一个查看器断开后自动暂停 CLI 串流

- 当某个会话的所有 Web 客户端正常断开、超时断开或写失败清理后
- Relay 会立即向对应 CLI 发送 `stop_capture`
- CLI 保持 WebSocket 在线，但暂停采集与编码，避免空转占用带宽
- 新查看器重新接入后，Relay 会再次发送 `start_capture`，串流自动恢复

### 3. 更稳健的查看器断开清理

- 多查看器场景下，只有最后一个查看器离开时才会暂停串流
- 写失败导致的僵尸查看器清理也会触发空闲检测
- 新增 Relay 集成测试覆盖“最后一个查看器断开才停止”和“断开后重新连接恢复串流”两条关键链路

### 4. 纳入上一轮未发布改动

- Desktop CLI 已支持通过 HTTP/HTTPS 代理连接 Relay，并自动适配 `ws://` / `wss://`
- 首页增加在线 Demo 与更清晰的下载入口
- macOS 下载链接在不同产物可用性下有更合理的回退行为

## 验证

本次发布前已完成以下验证：

- `go test ./cmd/client ./cmd/relay ./internal/relay`
- `go build ./cmd/relay`
- `go build -tags desktop -o /tmp/deskgo-desktop-stopcheck ./cmd/client`
- `docker compose up -d --build`
- 本地真实串流回归验证：
  - 查看器接入后可收到首帧
  - 最后一个查看器断开后 CLI 收到 `stop_capture`
  - 空闲期间 CLI 不再继续输出采集日志
  - 新查看器再次接入后串流恢复

## 已知限制

- Linux 输入控制当前仍要求 X11 / XWayland，纯 Wayland 暂未支持
- HEVC/H.265 仍未纳入正式产品能力，当前继续以 H.264 / JPEG 为正式支持范围
- Relay 仍建议部署在反向代理、隧道或访问控制层之后
