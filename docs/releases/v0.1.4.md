# DeskGo v0.1.4 release notes

## Release intent

`v0.1.4` is a focused homepage follow-up release. It restores a dedicated macOS Desktop CLI download block near the top of the landing page so Mac users no longer need to scroll down into the general downloads grid to find the correct build.

## Highlights

### 1. macOS download area is back at the top of the homepage

- the Chinese and English homepages now show a dedicated macOS quick-download block near the hero section
- both Intel (`AMD64`) and Apple Silicon (`ARM64`) downloads are shown immediately
- the quick block links back to the main downloads section so users can still reach the full platform matrix

### 2. End-user download flow is easier again

- Mac users can now grab the Desktop CLI from the first screen instead of hunting through the rest of the page
- the quick-download copy stays aligned with the end-user-focused homepage structure introduced in `v0.1.3`

## Validation

Validated before release with:

- `go build ./cmd/relay`
- local render verification for `/` confirming `macOS CLI 下载`, `下载 AMD64`, and `下载 ARM64`
- local render verification for `/en` confirming `macOS CLI download`, `Download AMD64`, and `Download ARM64`

## Known limitations

- this release only restores the top-of-page macOS download entry point; it does not change the underlying desktop/relay binaries
- relay deployments are still best placed behind a reverse proxy, tunnel, or another access-control layer
