# DeskGo v0.1.0 首发 Release Note（草案）

## 发布定位

这是 DeskGo 首个面向公开分发的桌面串流版本。
本次发布聚焦于一个更干净、更聚焦的产品形态：

- 原生桌面采集 Desktop CLI
- WebSocket Relay
- 浏览器查看器
- 多架构发布产物
- 双语网站入口

## 亮点

### 1. 多架构 Desktop CLI 与 Relay

首发即提供：

- Desktop CLI：macOS amd64/arm64、Windows amd64/arm64、Linux amd64/arm64/armv7/riscv64
- Relay：macOS amd64/arm64、Windows amd64/arm64、Linux amd64/arm64/armv7/riscv64/ppc64le/s390x

所有发布包都附带 `SHA256SUMS.txt`。

### 2. H.264 优先的桌面串流体验

- macOS 默认优先 H.264
- Linux 在检测到 ffmpeg/libx264 时默认优先 H.264
- 浏览器端自动在 H.264 / JPEG 之间协商与回退

### 3. 更完整的跨平台输入控制

- Windows：完成鼠标/键盘输入控制链路
- Linux：新增 X11/XTEST 输入控制
- macOS：修复按键释放事件

### 4. 更清晰的 Web 端体验

- CLI 断开时浏览器会主动收到 `session_ended` 提示并关闭连接
- 串流开始后自动隐藏页头页尾，优先展示桌面内容
- 首页与会话页同时提供中文与英文入口

### 5. 更适合分发的交付方式

- `./build.sh` 同时输出当前平台运行文件、发布包与 SHA256 校验文件
- Docker 镜像可直接承载网站与 `downloads/` 目录
- GitHub Actions 已加入构建产物与镜像发布流程

## 发布前清理

在首发前，仓库已经完成以下清理：

- 删除遗留试验代码、测试脚本与临时文档
- 清理调试页面与无关静态资源
- 修正仓库地址到 `topcheer/deskgo`
- 补齐中英双语核心文档

## 部署建议

推荐两种方式：

1. 直接使用 GHCR 镜像 `ghcr.io/topcheer/deskgo:latest`
2. 本地运行 `./build.sh` 后再用 Docker 构建镜像

如果需要公网访问，建议配合：

- Cloudflare Tunnel
- Nginx / Caddy 反向代理
- 企业 VPN 或访问控制层

## 已知限制

- Linux 输入控制当前要求 X11 / XWayland，纯 Wayland 还未纳入首发支持
- HEVC/H.265 仍是后续实验方向，首发只承诺 H.264 / JPEG
- Relay 当前不内置完整认证层，生产部署建议放在访问控制之后
