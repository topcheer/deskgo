# DeskGo v0.1.3 发布说明

## 发布定位

`v0.1.3` 的目标，是把最近一轮首页改版、Linux 输入修复、Wayland 自举能力，以及 Demo 域名切换真正打成可交付版本，让最终用户不必依赖尚未发布的 `master` 变更。

这次补丁版本主要聚焦四件事：

- 把首页改成真正面向最终用户的产品入口，而不是技术面板
- 在不破坏 Windows / macOS 逻辑的前提下，补强 Linux 鼠标和键盘控制
- 在 Wayland 场景下，尽量自动补齐 `ydotool` / `ydotoold`
- 把所有默认 Demo / 示例地址切换到 `deskgo.ystone.us`

## 亮点

### 1. 首页终于更像最终用户产品页

- 中英文首页都围绕最终用户的真实任务重新组织：
  - 在浏览器里打开桌面
  - 下载客户端
  - 配置自动运行
- 为 Shell / PowerShell 一键安装命令加入了快速复制按钮
- 把自建 Relay 和高级部署信息移出主路径，避免首页继续被技术细节淹没

### 2. Linux 输入控制明显更稳定

- 浏览器键盘事件现在会显式发送按下/抬起状态以及 `code`
- 鼠标按键位掩码现已正确对应浏览器里的左键 / 右键 / 中键语义
- 滚轮事件改成一次性 tap 风格的 4/5 号按钮注入，而不是粘滞状态模拟
- Linux 指针移动改为统一走 XTEST motion 注入
- 坐标映射现在会考虑显示器原点，修复偏移屏布局下的指针漂移
- 高频串流日志被压缩为周期性摘要，输入诊断日志更容易看清

### 3. Wayland 会话具备更好的自举能力

- 当检测到 Wayland 且系统缺少 `ydotool` / `ydotoold` 时，Linux 客户端现在会尝试通过以下包管理器自动安装：
  - `apt-get` / `apt`
  - `dnf`
  - `yum`
  - `pacman`
  - `zypper`
  - `apk`
- 自动安装支持：
  - 直接 root 执行
  - 无密码 `sudo` / `doas`
  - 桌面会话下的 `pkexec`
- 客户端现在还会自动探测或启动 `ydotoold`、选择可复用 socket 路径，并在 daemon 修复后重试失败的 ydotool 命令
- 这些修改都被限制在 Linux 输入后端内部，不会影响 Windows / macOS 的输入实现

### 4. Demo 与默认地址已切换到新域名

- Desktop CLI 的默认 Relay 地址现已改为 `wss://deskgo.ystone.us/api/desktop`
- README 示例、自动运行脚本默认值，以及代理相关测试都已同步切换
- 文档里展示的在线 Demo 地址现在统一指向 `https://deskgo.ystone.us`

## 验证

发布前已完成以下验证：

- `go test ./cmd/client ./cmd/relay ./internal/relay`
- `go test -tags desktop ./cmd/client`
- `go build ./cmd/relay`
- `go build -tags desktop -o /tmp/deskgo-client-macos ./cmd/client`
- `GOOS=windows GOARCH=amd64 CGO_ENABLED=0 go build -tags desktop -o /tmp/deskgo-client-windows.exe ./cmd/client`
- `GOOS=linux GOARCH=amd64 CGO_ENABLED=0 go build -tags desktop -o /tmp/deskgo-client-linux ./cmd/client`
- `GOOS=linux GOARCH=arm64 CGO_ENABLED=0 go build -tags desktop -o /tmp/deskgo-client-linux-arm64 ./cmd/client`
- `docker run --rm -v "$PWD":/src -w /src golang:1.24 go test -tags desktop ./cmd/client`
- `bash -n scripts/deskgo-autostart.sh`
- `scripts/deskgo-autostart.ps1` 的 PowerShell 语法解析校验
- 仓库全局搜索确认已不再存在 `deskgo.zty8.cn` 引用

## 已知限制

- Windows 桌面捕获 / 输入仍需要更多真实设备上的运行验证
- Wayland 原生控制仍取决于宿主机是否允许 `ydotool` / `ydotoold` 访问所需输入设备
- Relay 依然更适合部署在反向代理、隧道或其它访问控制层之后
