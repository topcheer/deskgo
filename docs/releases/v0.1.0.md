# DeskGo v0.1.0 first release notes (draft)

## Release intent

This is the first public release of DeskGo as a desktop streaming product.
The release is focused on a cleaner and more production-ready stack:

- native desktop capture via the Desktop CLI
- WebSocket relay delivery
- browser viewer
- multi-architecture release artifacts
- bilingual web entry points

## Highlights

### 1. Multi-architecture Desktop CLI and relay builds

The first release ships:

- Desktop CLI: macOS amd64/arm64, Windows amd64/arm64, Linux amd64/arm64/armv7/riscv64
- Relay: macOS amd64/arm64, Windows amd64/arm64, Linux amd64/arm64/armv7/riscv64/ppc64le/s390x

Every release bundle includes `SHA256SUMS.txt`.

### 2. H.264-first desktop streaming

- macOS defaults to the native H.264 path
- Linux defaults to H.264 when ffmpeg/libx264 is available
- the browser viewer negotiates H.264 / JPEG automatically and falls back when needed

### 3. Better cross-platform input control

- Windows now has end-to-end mouse and keyboard control
- Linux now includes X11/XTEST input control
- macOS key release handling is fixed

### 4. Clearer browser UX

- the viewer now receives `session_ended` and closes immediately when the CLI disconnects
- header and footer chrome are hidden while streaming is active
- both the landing page and session page are available in Chinese and English

### 5. Release-friendly delivery flow

- `./build.sh` emits host binaries, release artifacts, and SHA256 checksums together
- the Docker image can serve both the website and the `downloads/` directory directly
- GitHub Actions now cover artifact builds and container image publishing

## Pre-release cleanup

Before the first release, the repository was cleaned up to:

- remove old experimental code, test scripts, and temporary documentation
- remove diagnostic pages and unrelated static assets
- correct all repository references to `topcheer/deskgo`
- add bilingual core documentation

## Deployment guidance

Recommended paths:

1. use the published GHCR image `ghcr.io/topcheer/deskgo:latest`
2. run `./build.sh` locally and then build your Docker image from source

For public exposure, prefer one of the following:

- Cloudflare Tunnel
- Nginx / Caddy reverse proxy
- a VPN or other access-control layer in front of the relay

## Known limitations

- Linux input control currently requires X11 / XWayland; pure Wayland is not part of the first release scope yet
- HEVC/H.265 is still a future experiment; the first release guarantees H.264 / JPEG only
- the relay does not yet include a full built-in authentication layer, so production deployments should sit behind access control
