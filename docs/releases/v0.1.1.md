# DeskGo v0.1.1 release notes (draft)

## Release intent

`v0.1.1` is the first stability release after the initial launch. It focuses on real-world relay, browser viewer, and Desktop CLI lifecycle fixes discovered during end-to-end deployment validation.

This patch release primarily fixes two issues:

- the browser could connect without actually triggering `start_capture` on the CLI
- the CLI could keep streaming after every web viewer had already left, wasting bandwidth

It also rolls in the previously finished but not yet released improvements for:

- relay proxy support in the Desktop CLI
- the live demo and download entry points
- macOS download fallback behavior

## Highlights

### 1. Fix browser session injection so `start_capture` works again

- corrected the session page template bindings to avoid double-quoting the injected `session_id`
- the browser now connects to the real session instead of a quoted phantom session
- the relay can once again deliver `start_capture` to the actual CLI session

### 2. Pause CLI streaming when the last viewer disconnects

- when all web viewers disconnect cleanly, time out, or get removed after write failures
- the relay now sends `stop_capture` to the connected CLI immediately
- the CLI keeps its WebSocket session alive but pauses capture and encoding to save bandwidth
- when a new viewer arrives later, the relay sends `start_capture` again and streaming resumes automatically

### 3. Stronger viewer cleanup behavior

- multi-viewer sessions only stop streaming after the final viewer leaves
- stale viewer cleanup on broadcast failures now also triggers the idle check
- new relay integration coverage verifies both “stop only after the last viewer leaves” and “resume after reconnect”

### 4. Include the last unreleased follow-up changes

- the Desktop CLI can now connect to the relay through HTTP/HTTPS proxies with automatic `ws://` / `wss://` handling
- the landing page includes the live demo and clearer download entry points
- macOS download links now fall back more gracefully when a preferred artifact is unavailable

## Validation

Validated before release with:

- `go test ./cmd/client ./cmd/relay ./internal/relay`
- `go build ./cmd/relay`
- `go build -tags desktop -o /tmp/deskgo-desktop-stopcheck ./cmd/client`
- `docker compose up -d --build`
- a local end-to-end streaming probe verifying that:
  - a viewer receives the first frame after connecting
  - the CLI receives `stop_capture` when the last viewer disconnects
  - the CLI stays idle without continuing capture logs during the gap
  - a later viewer reconnect resumes streaming successfully

## Known limitations

- Linux input control still requires X11 / XWayland; pure Wayland is not in scope yet
- HEVC/H.265 is still not part of the supported product surface; H.264 / JPEG remain the supported codecs
- the relay should still be deployed behind a reverse proxy, tunnel, or other access-control layer
