# DeskGo v0.1.2 release notes

## Release intent

`v0.1.2` turns the recent relay, installer, and Windows startup fixes into a consumable release so the GitHub-hosted online installers and downloaded Desktop CLI binaries finally carry the same behavior as `master`.

This patch release focuses on three themes:

- making the cross-platform autostart installers production-safe
- fixing Windows-specific startup and PowerShell compatibility regressions
- normalizing relay/session handling so manual CLI usage is less fragile

## Highlights

### 1. Cross-platform autostart installers are now release-ready

- added guided and unattended installers for:
  - macOS / Linux: `deskgo-autostart.sh`
  - Windows PowerShell: `deskgo-autostart.ps1`
- installers download the matching Desktop CLI artifact plus `SHA256SUMS.txt`
- per-user desktop-session startup is used on every platform:
  - Windows Scheduled Task
  - macOS LaunchAgent
  - Linux XDG Autostart or optional `systemd --user`

### 2. Windows autostart is much safer

- fixed PowerShell encoding and Windows PowerShell 5.1 compatibility issues
- generated `run-desktop.ps1` launchers are now BOM-safe and parse correctly
- Windows startup now uses:
  - hidden PowerShell launch
  - delayed post-logon start
  - retry logging
  - cleanup of old tasks/processes during reinstall or uninstall
- launcher logs are written to `%LOCALAPPDATA%\DeskGo\logs\desktop.log`

### 3. Manual relay usage is more forgiving

- Desktop CLI now accepts relay URLs in either `http(s)://` or `ws(s)://` form and normalizes them before dialing
- session IDs are normalized case-insensitively across the client, relay, web pages, and installer-generated configs
- the relay/web URL helpers now preserve custom path prefixes correctly when deriving browser-facing session URLs

### 4. Online installer flow is aligned with GitHub usage

- the website, README files, and autostart docs now document GitHub Raw as the installer entry point
- cache-busting/no-cache fetch snippets reduce stale installer downloads
- the release bundle includes the installer scripts themselves so release assets and online docs stay aligned

## Validation

Validated before release with:

- `go test ./cmd/client ./cmd/relay ./internal/relay`
- `go build -tags desktop -o /tmp/deskgo-client-desktop ./cmd/client`
- `GOOS=windows GOARCH=amd64 CGO_ENABLED=0 go build -tags desktop -o /tmp/deskgo-client-windows.exe ./cmd/client`
- `bash -n scripts/deskgo-autostart.sh`
- `pwsh -NoProfile -File scripts/deskgo-autostart.ps1 install -RelayServer http://192.168.31.105:8082 -Session WiN11Studio -Version v0.1.1 -NonInteractive -DryRun`
- dry-run verification that installer-generated relay URLs normalize to `ws://` / `wss://`
- parser validation for the generated Windows `run-desktop.ps1` launcher

## Known limitations

- Windows autostart still supports `jpeg` only; Windows H.264 autostart encoding is not implemented yet
- Linux input control still depends on X11 / XWayland
- relay deployments are still best placed behind a reverse proxy, tunnel, or another access-control layer
