Install autostart online in one step.

The recommended flow is to copy the GitHub online install commands below. The shell and PowerShell installers support both guided prompts and unattended flags, detect the current OS/architecture, download the matching CLI, verify SHA256, write a fixed session config, and register per-user autostart.

macOS / Linux installer

Targets the current logged-in user session: LaunchAgent on macOS, XDG Autostart by default on Linux, with optional systemd --user.

  • Supports guided install/uninstall and --non-interactive flag-based execution
  • Lets you choose relay URL, fixed session name, codec, and release version
  • The install command pulls the script from GitHub Raw, while the CLI binary comes from the GitHub latest release by default
GitHub shell script Full guide 
rm -f /tmp/deskgo-autostart.sh
curl -fsSL -H 'Cache-Control: no-cache' "{{ .AutostartShellURL }}?ts=$(date +%s)" -o /tmp/deskgo-autostart.sh
bash /tmp/deskgo-autostart.sh install

Windows installer

Registers a per-user Scheduled Task at logon, which covers the usual desktop autostart flow without requiring administrator privileges.

  • Supports guided or unattended install, plus clean uninstall
  • Downloads the matching Windows CLI asset and verifies SHA256SUMS.txt
  • The recommended codec for Windows autostart is currently jpeg, and the install command also pulls the script from GitHub Raw
GitHub PowerShell script Full guide 
$script = Join-Path $env:TEMP 'deskgo-autostart.ps1'
$uri = "{{ .AutostartPowerShellURL }}?ts=$([DateTimeOffset]::UtcNow.ToUnixTimeSeconds())"
Remove-Item $script -ErrorAction SilentlyContinue
Invoke-WebRequest $uri -UseBasicParsing -Headers @{ 'Cache-Control' = 'no-cache' } -OutFile $script
powershell -ExecutionPolicy Bypass -File $script install

Before you roll it out

This section shows the user-facing GitHub online install flow directly; successful installation still depends on the release assets matching the script contract.

  • The latest release must contain the matching deskgo-desktop-<os>-<arch> asset and SHA256SUMS.txt
  • Autostart is intentionally scoped to the current logged-in desktop session, not a headless system service
  • The installer writes configuration to deskgo.json in the install root, so it also works with older release CLIs that do not understand -config

For predictable fleet rollouts, pin --version and review the build matrix, autostart guide, and GitHub repository.

Build and deliver in three steps.

Whether you are distributing clients or deploying the relay, the same build and access flow now applies.

1

Run the build script

Run ./build.sh to produce the current-platform runnable binaries, Desktop CLI downloads, relay release packages, and SHA256SUMS.txt.

2

Deploy the relay

Start the local binary or Docker image; the site will expose the downloads directory automatically and provide both / and /en entry points.

3

Run the CLI and share the link

Run the matching CLI build on the desktop host, then open the viewer with either the session ID or the generated direct link.
