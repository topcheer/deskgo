输入会话 ID，直接进入远程桌面。

适用于已经启动 CLI 的场景；浏览器将在会话页面等待客户端接入，并在桌面端断开时给出清晰提示。

CLI 启动后会在终端输出会话 ID 和访问链接；你也可以把链接直接分享给查看者。

等待接入时保持页面打开

当桌面端尚未连接时，浏览器会显示等待状态；接入成功后将自动切换到串流视图。

桌面端断开时自动提示

如果 CLI 退出、会话结束或连接异常，查看器会立即断开并显示恢复指引。

围绕稳定串流和一致体验做的核心优化。

从浏览器断线提示、串流页面布局，到 Docker 产物和 Windows CLI 一致性，DeskGo Relay 现在更像一个可交付的平台入口。

断线感知更明确

CLI 会话结束时，Relay 主动通知并关闭查看器连接，避免 Web 端停留在无响应状态。

  • 断开即提示
  • 区分会话结束与网络问题
  • 支持手动重连

串流视图更专注

开始串流后自动隐藏页头与页尾，把更多空间留给桌面内容；错误或断开时再恢复状态信息。

  • 观看区域最大化
  • 状态栏自动恢复
  • 减少操作干扰

跨平台分发更顺手

Docker 镜像内同时产出 Relay 和 CLI 下载物，Windows CLI 与 macOS 维持统一会话启动体验。

  • 镜像内含下载产物
  • Windows 构建可验证
  • CLI 行为更一致

三步完成接入。

无论是本地调试还是 Docker 部署，都可以沿用同一套操作路径。

1

部署 Relay

通过 Docker 或本地二进制启动 Relay，获得 Web 界面、会话页和 CLI 下载入口。

2

启动 CLI

在桌面主机上运行 CLI，终端会输出会话 ID、串流编码状态和访问链接。

3

浏览器接入

输入会话 ID 或直接打开访问链接，查看器会自动协商编码并进入串流模式。
