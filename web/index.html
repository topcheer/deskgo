在线一键安装自动运行。

推荐直接复制下方 GitHub 在线安装命令。Shell / PowerShell 脚本同时支持引导式安装与非引导参数模式，会按当前设备平台下载匹配 CLI、校验 SHA256、写入固定 session 配置，并按当前用户注册自动启动。

macOS / Linux 安装器

面向当前登录用户：macOS 使用 LaunchAgent，Linux 默认使用 XDG Autostart，并支持切换到 systemd --user。

  • 支持交互式安装 / 卸载，也支持 --non-interactive 直接传参
  • 支持自定义 Relay、固定 Session 名字、Codec 与版本
  • 安装命令直接从 GitHub Raw 拉取脚本，CLI 二进制默认来自 GitHub latest release
GitHub shell 脚本 查看完整说明 
curl -fsSL {{ .AutostartShellURL }} -o /tmp/deskgo-autostart.sh
bash /tmp/deskgo-autostart.sh install

Windows 安装器

使用当前用户的 Scheduled Task 在登录后启动 CLI，无需管理员权限即可覆盖大多数桌面自启动场景。

  • 支持引导式或一次性传参安装，也支持卸载清理
  • 自动下载匹配架构的 Windows CLI，并验证 SHA256SUMS.txt
  • Windows 当前自动运行模式建议使用 jpeg codec，安装命令同样直接从 GitHub Raw 拉取脚本
GitHub PowerShell 脚本 查看完整说明 
$script = Join-Path $env:TEMP 'deskgo-autostart.ps1'
Invoke-WebRequest '{{ .AutostartPowerShellURL }}' -OutFile $script
powershell -ExecutionPolicy Bypass -File $script install

使用前请确认

这里展示的是直接面向用户的 GitHub 在线安装方式；真正安装成功仍取决于发布产物是否满足脚本约定。

  • latest release 必须包含匹配平台的 deskgo-desktop-<os>-<arch> 资产与 SHA256SUMS.txt
  • 自动运行针对“当前登录用户的桌面会话”，不适合无人登录的系统服务场景
  • 脚本默认把配置写到安装目录中的 deskgo.json，因此也兼容尚未支持 -config 的旧 release CLI

如果你要做可预测的批量部署，请固定 --version，并参考 构建矩阵、自动运行文档 与 GitHub 仓库。

三步完成构建与接入。

无论你要分发客户端还是部署 Relay，都可以沿用同一套构建与访问路径。

1

运行构建脚本

执行 ./build.sh，生成当前平台运行文件、Desktop CLI 下载包、Relay 发布包与 SHA256SUMS.txt。

2

部署 Relay

启动本地二进制或 Docker 镜像；站点会自动从 downloads 目录提供可下载构建，并支持 / 和 /en 两个入口。

3

启动 CLI 并分享链接

在桌面主机上运行对应架构的 CLI，随后通过会话 ID 或访问链接进入浏览器查看器。
